# Project Roadmap

本文档定义项目固定开发路线。AI 不得擅自重排阶段顺序。开发顺序必须遵循 MVVM 分层依赖：`Shell -> Domain -> Infrastructure -> Application -> Presentation -> Platform Integration -> QA/Release`。

## 阶段 0：文档与接力规则

- 状态：`complete`
- 入口条件：无
- 必须产出：
  - 用户需求文档
  - 总设计文档
  - 开发任务拆解
  - 接力开发文档体系
- 退出条件：
  - 无记忆 AI 可以仅凭文档确定当前任务

## 阶段 1：解决方案与应用壳层骨架

- 状态：`complete`
- 入口条件：
  - 阶段 0 完成
- 必须产出：
  - Uno Platform 客户端项目骨架
  - ASP.NET Core 服务端项目骨架
  - 5 页应用壳层
  - 默认启动进入主页
  - 底部导航与基础路由
- 禁止提前做：
  - 复杂业务页面逻辑
  - AI 请求逻辑
  - 平台特性实现
- 退出条件：
  - 客户端和服务端都能启动最小骨架
  - 应用壳层可切换五个主页面占位

## 阶段 2：Domain 层建模

- 状态：`complete`
- 入口条件：
  - 阶段 1 完成
- 必须产出：
  - 事项、设置、聊天记录、同步变更模型
  - 时间块规则
  - 重复规则
  - 提醒规则
  - 主页命中与重叠计算规则
- 退出条件：
  - 领域模型和规则可被 Application 层直接复用

## 阶段 3：Infrastructure 层

- 状态：`complete`
- 入口条件：
  - 阶段 2 完成
- 必须产出：
  - SQLite 本地数据层
  - 服务端 PostgreSQL 数据层
  - 认证 API 与存储
  - 远程同步 API 客户端/服务端接口
  - 日志
  - 通知抽象
  - 小组件快照抽象
- 退出条件：
  - 本地与远程基础设施可被 Application 层调用

## 阶段 4：Application 层

- 状态：`complete`
- 入口条件：
  - 阶段 3 完成
- 必须产出：
  - 认证用例
  - 事项 CRUD 用例
  - 设置读写用例
  - 自动同步编排
  - 同步状态机与冲突收敛
  - 列表筛选/排序/重排用例
  - AI 请求与事项摘要编排
  - 主页布局计算服务
- 退出条件：
  - 所有关键业务流程都可通过服务和用例层调用

## 阶段 5：Presentation 基础页面

- 状态：`complete`
- 入口条件：
  - 阶段 4 完成
- 必须产出：
  - 登录页
  - 设置页主结构
  - 添加/编辑页
  - 事项详情组件
- 退出条件：
  - 用户可登录
  - 用户可新增、编辑、查看三类事项
  - 设置页主结构可用

## 阶段 6：Presentation 主页

- 状态：`complete`
- 入口条件：
  - 阶段 5 完成
- 必须产出：
  - 时间选择组件
  - 主页周/月视图
  - 时间块网格
  - 跨格渲染
  - 重叠透明度
  - 长按/点击/命中规则
- 退出条件：
  - 手机端主页主流程可用
  - 宽屏月视图能力可用

## 阶段 7：Presentation 列表页

- 状态：`complete`
- 入口条件：
  - 阶段 5 完成
- 必须产出：
  - 标签页
  - 排序
  - 手动重排
  - 主题切换
  - 更多设置入口
  - 滑动编辑删除
  - 浮动添加按钮默认填充
- 退出条件：
  - 列表页主流程可用
  - 列表页设置能力符合原始需求

## 阶段 8：Presentation AI 页

- 状态：`complete`
- 入口条件：
  - 阶段 5 完成
  - Application 层 AI 编排完成
- 必须产出：
  - AI 设置页
  - AI 聊天页
  - 聊天记录按日存储
  - 按日/周/月展示聊天记录
  - 自然语言增删查事项
- 退出条件：
  - AI 页满足首版验收样例

## 阶段 9：实时同步与状态一致性

- 状态：`complete`
- 入口条件：
  - 阶段 5 完成
- 必须产出：
  - 自动后台同步
  - 手动同步入口
  - 事项自动收敛
  - 设置自动收敛
  - 同步状态展示
- 退出条件：
  - 多设备在不手动触发的情况下可收敛事项和设置

## 阶段 10：平台集成

- 状态：`complete`
- 入口条件：
  - 阶段 6、7、8、9 完成
- 必须产出：
  - 本地通知平台映射
  - 四类小组件平台映射
  - 平板横竖屏适配
  - Windows 适配
  - Web 适配与降级
- 退出条件：
  - 手机、平板、Windows、Web 主流程可用
  - 各平台通知和小组件能力可用或有明确降级策略

## 阶段 11：QA、验收、发布收尾

- 状态：`active`
- 入口条件：
  - 阶段 1 到 10 完成
- 必须产出：
  - 自动化测试补齐
  - 性能验证
  - 原始需求映射验收
  - 文档与代码一致性收尾
  - git 提交与推送，或记录最终阻塞
- 退出条件：
  - `PROJECT-ACCEPTANCE.md` 全部通过

## 里程碑

### 里程碑 A：MVP

- 包含：阶段 1 至 7
- 目标：手机端具备可登录、可新增事项、可使用主页与列表页的完整主流程
- 状态：`complete`

### 里程碑 B：AI 与实时同步

- 包含：阶段 8 至 9
- 目标：AI 页可自然语言增删查事项，且事项与配置支持自动同步收敛
- 状态：`complete`

### 里程碑 C：平台闭环

- 包含：阶段 10
- 目标：通知、小组件、多平台主流程完成
- 状态：`complete`

### 里程碑 D：发布就绪

- 包含：阶段 11
- 目标：验收、测试、性能、文档全部闭环，并完成 git 仓库收尾上传
- 状态：`active`
